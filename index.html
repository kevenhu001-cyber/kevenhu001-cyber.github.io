我想做能解决真实问题的东西。

过去几年，我在AI 应用、Web 全栈 和 Android 原生 三个方向都做过完整的项目——每个都是从 0 到 1、落到可部署服务的产品级工程。

我相信好的软件像好的工具：克制、好懂、能用。 每个项目都由一个真实的问题驱动设计，而不是为了技术而技术： 一个 AI 学习伴侣该像老师一样引导而非直接给答案； 一个文件收集工具应该让老师三分钟上线、剩下的交给系统； 一个题库不该只是题目列表，而是要真正帮学生理解每个公式。

苏格拉底式的 AI 老师

Socrates — AI learning companion

核心理念：学习是苏格拉底式的行为，不是问答行为。基于这一洞察，Socrates 把 LLM 塑造为一位会耐心引导的「老师」——先用例子再讲概念、用引导式提问代替直接否定，而不是扮演一个什么都知道的「助手」。

自研流式渲染引擎：不同于传统 chat UI 在模型输出完才一次性渲染（loading → 全部出现的跳变感），Socrates 实现了逐行状态机 Markdown 渲染器——# → <h1> 在模型刚打出 # 的同一帧就出现，列表、引用、代码围栏全部边输出边渲染，零闪烁。完成后才走完整的 marked + KaTeX + highlight.js 管线。这是目前为数不多实现真正「渐进式渲染」的 AI 聊天产品之一。

推理模型思维链折叠：DeepSeek R1、QwQ 等模型的 <think>...</think> 思维链被渲染为可折叠的 <details> 卡片，思维链前后的内容仍然正常展示，不打断阅读流。

Viz 交互画布：对话框输入 ```html 或 ```viz，模型输出即渲染为沙箱隔离的交互式 iframe（sandbox="allow-scripts"，不授 allow-same-origin）。流式输出期间显示加载卡片，用户永远不会看到原始代码闪烁。

苏格拉底导师模式：内置诊断阶段评估学习者知识边界 → 生成结构化教学计划（Motivate → Define → Develop → Illustrate → Exercise → Check）→ 知识边界图谱可视化 → 错题本自动捕获与分类 → 卡壳检测与逐级干预（温和提示 → 双选项 → 四选项）。系统提示词本身就是一个产品功能。

Agent 工具模式：SSE 代理执行，支持 web_search（实时网页结果）、code_interpreter（Pyodide WASM Python 沙箱）、文件 I/O 等多种工具。

后端子系统和路由：Express 5 + PostgreSQL + Drizzle ORM，覆盖 30+ API 路由：聊流 SSE、会话 CRUD、项目文件夹、标签筛选、Cmd-K 全局搜索（fuse.js）、斜杠命令面板、错题本、知识边界、跨会话记忆、公共分享链接、用量追踪与计费看板、团队工作空间、推送通知（FCM）、语音转录、图片识别、文档提取（PDF/DOCX/XLSX/PPTX/EPUB/RTF）、网页搜索（MiniMax + Bing + SearXNG 三引擎竞速降级）、Prompt 模板管理。用户 API Key 加密静态存储（AES-GCM），只在单次请求内存中持有。

Android 原生客户端：Kotlin + Jetpack Compose + Material 3，原生 SSE 流式对话、深色/浅色双主题、Room 离线缓存、知识图谱与错题本同步、Deep Link 深度链接、GitHub Actions APK 自动化构建。

安全设计：Cookie 认证（bcrypt + 64 字符随机 sid，HttpOnly + SameSite=Lax）、双重提交 CSRF 防御、SSRF 防护（fetch-batch 逐跳验证重定向）、速率限制、Helmet 安全头。

极简文件收集平台

File-Collect — painless file submission

解决的问题：微信群收文件——同名文件会被覆盖、改名丢失来源、收集完成后要一个个点开保存。File-Collect 让教师/组织者只需要三件事就能上线收集：登录后台 → 新建收集任务 → 把链接发给参与者。剩下的事情全部由系统搞定。

多主题并行收集：同一组织者可同时拥有多个收集主题（如"建模比赛"、"实验报告"、"毕业设计"），每个主题拥有独立提交链接、独立存储空间和独立的配置（暂停/恢复、截止时间、速率限制）。

管理后台：完整的单页管理界面——密码登录（默认 000000，可环境变量配置）生成 32 字节随机 hex token -> 持久化到 config/sessions.json，服务器重启后登录状态不丢失。后台功能包括：查看所有提交者清单、按用户筛选文件（含文件大小显示）、单个删除、批量删除用户（二次确认）、单用户文件 ZIP 打包下载、全收集 ZIP 打包下载、新建/删除收集主题（删除前安全检查——已有文件则拒绝）、配置收集暂停/恢复与截止时间（前端实时倒计时）。

大文件上传：单文件最大 500MB，支持拖拽上传，原始文件名 UTF-8 解码（multer binary → Buffer → utf8 转换），中文文件名不乱码。

速率限制与安全：IP 级别上传频率控制（可配置每小时每 IP 上限），纯内存 Map 实现 + 每 5 分钟自动清理过期记录。Nginx 层配合长 timeout（86400s）保证大文件不中断。

工程架构：Node.js 22 + Express 4 + Multer + Archiver（最高压缩级别 9 流式 ZIP，无临时文件）。全应用无数据库——配置存 JSON 文件、会话存 JSON 文件、上传文件存文件系统，原子写入保证一致性。前端全原生 HTML + CSS + JavaScript（零框架、无构建步骤）。部署链路：Nginx 反代 + Let's Encrypt SSL + PM2 进程管理。

大学生自建题库

加成学习 — LaTeX-rendered question bank

为什么做：学期末刷题时，纸质题目翻不到、Ctrl+F 不支持数学公式、商业题库要充值还有广告。一个学生自己上传、整理、刷题；老师一键出卷；公式漂亮渲染；历年卷 PDF 自动转结构化题目——这就是加成学习要做的事。

四类题型完整支持：选择题（选项 JSON 存储，自动批改归一化对比）、填空题（管道符 | 分隔多个可接受答案）、判断题（兼容 正确/对/是/T/True 等多种输入）、简答题（含子题格式化，标记为待人工批改）。难度 1-5 级分层。

LaTeX 渲染管线：自研预处理层——剥离 LaTeX 导言命令、归一化 \(...\) → $...$、处理常见物理单位（\muC、\kV 等）、黑板粗体与花体字母。然后走 remark-math + rehype-katex 水管，渲染失败时 fallback 显示源码。编辑器中内置 LaTeX 工具栏，一键插入数学符号。

AI 多引擎支持：支持 10+ LLM 提供商、约 40 个模型——OpenAI（GPT-4o/4o-mini/4-Turbo/3.5-Turbo/5.2-Thinking）、阿里千问（Qwen-Max/Plus/Turbo/Qwen3-Max-Thinking）、腾讯混元（Pro/256K/Standard/Lite）、DeepSeek（Reasoner/Chat/Coder）、Google Gemini（2.0 Flash/1.5 Pro/1.5 Flash/3.0 Flash/3 Pro）、Anthropic Claude（3.5 Sonnet/3 Opus/3 Haiku/Opus 4.6）、MiniMax、xAI Grok、SiliconFlow 聚合平台、以及自定义 OpenAI 兼容端点。AI 功能包括：智能出卷（选章节/题型/难度/题量 → AI 生成）、题目详解、图片 OCR 识别、PDF 题目识别。

试卷生成与考试系统：PaperBuilder 拖拽组卷界面——搜索筛选题库 → 拖入试卷区 → 拖拽排序 → 每题独立赋分 → 总分配置 → 预览保存。考试端支持计时限制（到期自动交卷）、逐题导航（进度条 + 题号网格）、答案输入因题型而异、简答题附件上传（图片/PDF/文档）、交卷后自动批改展示得分与错题分析。

PDF 生成服务：双实现并行——主链路 Python Flask + Playwright（无头 Chromium 渲染含 KaTeX 公式的 HTML 到 A4 PDF，含页眉/页脚/分页/题号/答案区），备用链路 Node.js + pdf-lib（含完整的 LaTeX 到纯文本转换管线：希腊字母、矩阵、分数、积分、集合论符号等）。PDF 导入走 pdf-parse + pdf2pic 转图片 + AI 视觉模型识别结构化题目。

用户系统：邮箱验证注册（6 位验证码）、登录速率限制 + 超出显示验证码、5 次失败锁定 30 分钟、密码强度校验、忘记/重置密码、记住我 30 天 token、登录审计日志、个人资料编辑/改密/注销。

技术架构：三服务 monorepo——React 18 + Vite 5 + TypeScript + Zustand + Tailwind + framer-motion 前端（:3000 代理到 :3001）、Express + Prisma + SQLite 后端（30+ API 路由，JWT + bcrypt + Helmet + rate-limit 中间件栈）、Python Flask + Playwright PDF 服务（端口 5000）。一键部署脚本 deploy.ps1 全自动安装构建。

贝克兰德的 AI 文字冒险

神秘之旅 — AI-driven RPG in Victorian gothic world

世界观：1349 年的贝克兰德（伦敦）。你扮演一名霍伊大学的历史系学生，意外卷入了超凡力量的世界，通过"占卜家"序列逐步揭开真相。基于《诡秘之主》原著世界观的克苏鲁风格文字冒险，故事不是写死的——AI 来当 DM，每次开档、每个选择都能给出不一样的展开。

双模态状态解析引擎：AI 输出中的结构化标签 {spirituality:80/100} {madness:15} {location:贝克兰德广场} {inventory:+钥匙,-学生证} 等被正则精确提取。当 LLM 偶尔漏输标签时，NLP 启发式解析兜底——通过关键词检测"灵性消耗"、"理智下降"等描述，配合程度副词（轻微/中等/严重）推算增量变化。两种结果合并为 StateUpdate 输出，附带 0-100 的一致性感评分。

RPG 状态系统（38 个字段）：灵性（当前/上限）、疯狂值（数值 + 腐化层级）、金钱（金镑/先令/便士三币制）、背包物品列表、当前序列途径与序列等级（0-9）及消化进度、已掌握能力、知识库、状态效果、阵营声望映射、NPC 关系网、命运织网节点（关键选择/物品接触/NPC 关系，持久化可回溯触发）、封印物（含各自人格状态与效果列表）、灵视开关、角色扮演进度、疯狂表现阶段（正常→内心独白→幻影同伴→失控）、日常日志、称号、成就树、命运路径、神秘点数、进度指标。

世界状态：时间（精确到小时）、地点、天气、已访问位置集合、进行中任务列表、NPC 状态追踪。

22 条完整序列途径：占卜家、偷盗者、学徒、歌颂者、水手、秘祈人、阅读者、律师、罪犯、猎人、窥秘人、耕种者、收尸人、不死者、通识者、巫师、悖论途径、愚者、暴君、太阳、上帝、母神——每条途径含 10 个完整序列等级、关键能力、对应源质与外神，总计 220 个序列等级数据。

12 个开局剧本：含难度分层（EASY ×2、NORMAL ×6、HARD ×2、NIGHTMARE ×2）和标签系统（如"占卜家"、"原初航线"、"新手友好"），以及 4 个彩蛋剧本（扮演外神、造物主等，开局属性 9999/9999）。

AI 后端多引擎：四模式运行时切换——DeepSeek V3.2（SiliconFlow 渠道，默认）、Doubao Mini（火山引擎）、Minimax M2.5、以及任意 OpenAI 兼容的自定义端点。所有引擎共享可配置参数：temperature 0.8、max_tokens 8192、top_p 0.95、frequency_penalty 0.2、presence_penalty 0.2。参数可通过设置界面随时调整。

上下文自动压缩：历史 ≤ 10 条全部保留；总字符 > 8000 时保留最近 6 条 + 老消息摘要（提取关键位置标签和叙事片段）；中间地带保留最近 12 条。确保长对话不超 token 限制且故事连贯。

UI/UX：暗色哥特主题（黑底金饰）、状态栏实时动画条（灵性蓝条、疯狂红条、金币/时间显示）、打字机逐字叙事 + 实时尾部滚动、AI 生成选择按钮的渐入缩放动画、流式响应加载齿轮动画 + 动态点、游戏中一键切换 AI 模型、10 页世界观引导教程（可跳过）、20 个存档位（DataStore JSON 持久化，含元数据：角色名/地点/灵性/时间戳）。

技术架构：Kotlin 1.9.22 + Jetpack Compose（BOM 2024.02）+ Material 3 + MVVM（StateFlow）+ OkHttp 4.12（原生 SSE 流式解析）+ Gson 序列化 + Jetpack DataStore 持久化。AGP 8.13.2 构建，minSdk 26，targetSdk 34。APK 约 11.7 MB。
