苏格拉底式的 AI 老师

重新定义 AI 聊天界面——从文本框与日志，到一个会耐心引导你思考的老师。

核心理念：「学习是苏格拉底式的行为，不是问答行为。」Socrates 将 LLM 塑造为一位会耐心引导的「老师」，而非什么都知道的「助手」。系统提示词本身就是产品功能——改为"老师"而非"助手"后，整个对话的语调、用词、节奏完全变了。

自研逐行状态机 Markdown 渲染引擎：Socrates 独创的 formatMsgProgressive 函数每帧驱动一个状态机——# 打出即渲染为 <h1>、列表/引用/代码围栏全部边输出边渲染，不依赖任何第三方库（无 marked、无 flicker）。输出完成后走完整的 formatMsg 管线（marked + KaTeX + highlight.js + viz 沙箱 + 思维链折叠）。这是目前为数不多实现真正"渐进式渲染"的 AI 聊天产品之一。

推理模型支持：DeepSeek R1、QwQ 等模型的 <think>...</think> 思维链折叠为可展开的 <details> 卡片，链前链后内容正常展示，不打断阅读流。

Viz 交互画布：输入 ```html 或 ```viz，模型输出即渲染为沙箱隔离的 iframe（sandbox="allow-scripts" 禁用 same-origin），流式期间显示加载卡片，用户永远不会看到原始代码闪烁。

苏格拉底导师模式：诊断阶段评估知识边界 → 结构化教学计划（Motivate → Define → Develop → Illustrate → Exercise → Check）→ 知识边界图谱可视化 → 错题本自动分类 → 卡壳检测与逐级干预（温和提示 → 双选项 → 四选项）。

Agent 工具模式：SSE 代理执行——typing 事件实时显示思考状态、tool_use 事件触发 web_search（三引擎竞速降级：MiniMax → Bing → SearXNG）/ code_interpreter（Pyodide WASM 池预热启动，避免冷启动 3-5s 延迟）/ 文件 I/O。服务端 30s drain timeout 保证活跃 SSE 流不中断。

后端子系统和路由（30+ API）：Express 5 + PostgreSQL + Drizzle ORM。聊流 SSE 端点、会话 CRUD、项目文件夹拖拽排序、标签多选筛选、Cmd-K 全局搜索（fuse.js 跨会话标题+内容）、斜杠命令面板、错题本 CRUD、知识边界节点聚合、跨会话记忆读写、公共分享链接（/s/:token）、用量追踪与计费看板、团队工作空间、FCM 推送通知、语音转录、图片识别、文档提取（PDF/DOCX/XLSX/PPTX/EPUB/RTF）、Prompt 模板管理。用户 API Key 加密静态存储（AES-GCM 派生自 SESSION_SECRET），只在单次请求内存中持有，绝不落日志。CSRF 防御有三条独立代码路径处理不同的隐私模式边界条件。SSRF 防护使用 redirect:'manual' 逐跳验证而非跟随重定向。

Android 原生客户端：Kotlin + Jetpack Compose + Material 3，双主题配色同步 Web 端。原生 SSE 流式对话、OkHttp + StateFlow 管线。知识图谱与错题本 Room 离线缓存。Deep Link + App Links 验证邮件深度链接。CI 自动构建 debug + release APK。

前端架构（~13,824 行 main.js）：双渲染管线（渐进式流渲染 + 最终全渲染）、Proxy 响应式状态、国际化 i18n、CDN 资源加载管线（marked + KaTeX + highlight.js + mermaid + fuse.js 延迟加载）、自定义 Vite 插件解决模块依赖顺序问题。Nonce 占位符转义方案避免 naive escape → markup → restore 在用户输入含 <code> 字面量时出错。

产品上下文自动刷新：后台每 6 小时抓取官网内容生成产品知识上下文块注入系统提示词，让模型能回答产品定价/功能/FAQ 等问题。

2025 — 至今 · 已部署上线 核心栈：Express 5 · PostgreSQL · Drizzle ORM · Kotlin Compose 3 · Vanilla JS + Vite · Pyodide WASM

为什么做这个

大多数 AI 聊天产品的 UX 都很相似 —— 一个文本框、一段日志。 但我想要的不是 Q&A，我想要一个会引导我思考的老师。

这个项目的核心 insight 在于：学习是苏格拉底式的行为，不是问答行为。 好的老师不直接给答案，而是先用例子、再讲概念；用引导式提问代替"你错了"。

"就像一位耐心温和的老师在跟你聊天。平时正常对话，不刻意教东西。 对方问问题或遇到困难时，再拿出老师的引导感； 讲东西时先举例子再讲概念，用'换个角度想想'代替'你错了'。"

这是项目内置的 prompts/teacher-mode.md 系统提示词原文。 这个 prompt 就是 Socrates 区别于其他 AI 聊天产品的灵魂。

核心功能

逐行 Markdown 渲染

传统 chat UI 在 LLM 输出完才一次性渲染，会有明显的"loading → 全部出现"的跳变感。 Socrates 自己实现了一个逐行状态机 Markdown 渲染器： # → <h1> 在模型刚打出 # 的同一帧就出现； 列表、引用、代码围栏、行内 **bold** 全部边输入边渲染。 完成后才走完整的 marked + KaTeX + highlight.js 管线。整个过程零闪烁。

推理模型支持

DeepSeek R1、QwQ 等推理模型输出形如 <think>...</think> 的思维链。 Socrates 把它折叠为 <details> 卡片，思维链前后的内容仍然正常展示。

可交互 Viz 画布

对话框里输入 ```html 或 ```viz，模型就会输出一个可交互的 React/HTML/SVG 作品，沙箱隔离在同一个源的 iframe 里。

文档解析

上传 PDF / DOCX / XLSX / PPTX / EPUB / RTF 文件，服务端解析成纯文本注入上下文。

组织功能

项目（对话文件夹）+ 独立 System Prompt、多标签筛选、置顶归档、30 天回收站、Cmd-K 全局搜索、斜杠命令面板。

安全模型

  • 认证：bcrypt 哈希 + 64 字符随机 sid cookie，HttpOnly + SameSite=Lax
  • CSRF：双重提交 cookie 模式
  • API Key：加密静态存储；浏览器只看到掩码；代理只在单次请求内存中持有
  • Viz iframe：sandbox="allow-scripts"，不授 allow-same-origin

学到了什么

  1. 流式 UI 的关键是"渐进式渲染"。逐行状态机虽然写起来繁琐，但它给了用户"AI 在我面前思考"的真实感。
  2. 系统提示词本身就是一个产品功能。改成"老师"而非"助手"的几个字差异，让整个对话的语调、用词、节奏完全变了。
  3. 不要相信模型。每帧跑一个 stripChatArtifacts() 函数过滤模板字符。
  4. 用 nonce 占位符做转义。朴素的"escape → markup → restore"在用户文本里出现 <code> 字面量时会炸。
