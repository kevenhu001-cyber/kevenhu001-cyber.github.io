大学生自己的题库

纸质题目难检索、Ctrl+F 不支持数学公式、商业题库要充值还有广告——那不如自己做一套。

完整的刷题与出卷平台：一个学生自己上传、整理、刷题；老师一键出卷；公式漂亮渲染；历年卷 PDF 自动转结构化题目。

四类题型：选择题（选项 JSON 存储，批改归一化字符对比）、填空题（管道符 | 分隔多个可接受答案）、判断题（兼容 正确/对/是/T/True 等多种输入）、简答题（含子题序号格式化，标记待人工批改）。难度 1-5 分层。

LaTeX 渲染管线：自研预处理层——剥离 LaTeX 导言命令、归一化 \(...\)→$...$、处理物理单位（\muC、\kV）、黑板粗体与花体字母。而后走 remark-math + rehype-katex 水管，渲染失败时 fallback 显示源码。编辑器内置 LaTeX 工具栏一键插入数学符号。

AI 多引擎系统（10+ 提供商 / ~40 模型）：OpenAI（GPT-4o/4o-mini/4-Turbo/5.2-Thinking）、阿里千问（Qwen-Max/Plus/Qwen3-Max-Thinking）、腾讯混元（Pro/256K/Standard）、DeepSeek（Reasoner/Chat/Coder）、Google Gemini（2.0 Flash/1.5 Pro/3.0 Flash/3 Pro）、Anthropic Claude（3.5 Sonnet/3 Opus/Opus 4.6）、MiniMax、xAI Grok、SiliconFlow 聚合（Qwen2.5/Llama 3.1/DeepSeek 等开源模型）、自定义 OpenAI 兼容端点。AI 能力：智能出卷（选章节/题型/难度/题量 → AI 生成 → 预览/编辑/确认）、题目详解（流式返回可编辑保存）、图片 OCR 识别（base64 → 视觉模型 → 结构化题目 JSON）、PDF 题目识别（pdf-parse + pdf2pic 转图片 → AI 视觉识别）。

试卷生成与考试系统（完整闭环）：PaperBuilder 拖拽组卷——搜索筛选题库 → 拖入试卷区 → 拖拽排序 → 每题独立赋分 → 总分配置 → 草稿自动保存到 localStorage → 预览确认保存。考试端：计时限制（到期自动交卷）、逐题导航（进度条 + 题号网格 + 快速跳转）、答案输入因题型自适应、简答题附件上传（图片/PDF/文档）、交卷后自动批改展示得分与错题分析。

PDF 生成双实现：主链路 Python Flask + Playwright（无头 Chromium 渲染含 KaTeX 公式的 HTML 到 A4 PDF，带页眉/页脚/分页/题号/答案区）。备用链路 Node.js + pdf-lib（含完整 LaTeX→纯文本转换管线：希腊字母、矩阵、分数、积分、集合论、化学式等符号映射）。

用户系统：邮箱验证注册（6 位码）、登录速率限制 + 5 次失败锁定 30 分钟 + 验证码保护、密码强度校验、忘记/重置密码、记住我 30 天 token、登录审计日志、个人资料编辑/改密/注销。

技术架构（三服务 monorepo）：前端 React 18 + Vite 5 + TypeScript + Zustand + Tailwind + framer-motion（13 个页面路由，KaTeX+react-markdown 渲染管线，AI 配置与模型选择面板）。后端 Express + Prisma + SQLite（30+ API 路由，JWT + bcrypt + Helmet + rate-limit 中间件栈，PDF 识别/生成/导入服务）。PDF 服务 Python Flask + Playwright（独立端口 5000）。一键部署脚本 deploy.ps1 全自动安装构建。

2025 · Full-stack 核心栈：React 18 · Vite · TypeScript · Tailwind · Prisma · SQLite · KaTeX · Playwright · Flask

为什么做这个

学期末刷题时，能用的工具总是差点意思：

  • 纸质题目——翻不到、印刷差、想按章节筛做不到
  • Word / PDF 流传——Ctrl+F 不支持数学公式、LaTeX 渲染失败
  • 商业题库 App——题目不全、要充值、还附带广告

我想要的是：每个学生自己上传、整理、刷题；老师一键出卷；公式漂亮呈现；历年卷 PDF 自动转结构化题目。

核心特性

LaTeX 公式原生渲染

数学题最大的痛点。用 remark-math + rehype-katex 全链路打通， $E = mc^2$ 和 $$\int_0^1 x^2 dx$$ 都能正常显示。

AIGC 智能出卷

选好章节、题型、难度、题量 → AI 自动出题 → 后端入库 → 后台可以预览、调整或整卷替换。

PDF 自动导入

上传历年卷 PDF，服务端用 pdf-parse + 自研解析器， 把试卷拆成结构化题目。渲染时附带原始版面图方便核对。

试卷生成器 & 在线做题

从题库挑选题目、自由排版、一键导出 PDF。在线做题支持单题练习、整卷模拟、错题重做。 每做完一份卷子自动统计正确率、用时、知识点分布。

学到了什么

  1. 富文本 + 数学公式是个无限坑。不要相信任何单一 Markdown 库能搞定，必须自己写"清洗 → 分词 → 行内 / 块级判定"的预处理层。
  2. PDF 是个比 Markdown 更脏的格式。从 Word 导出 PDF、从 LaTeX 编译 PDF、扫描件 PDF 完全是三种文档，需要不同的解析策略。
  3. Prisma 比裸 SQL 开发快得多，但写复杂查询时还是得回到 $queryRaw。
  4. AI 生成的内容必须有二次编辑界面。Preview + Edit + Confirm 是 AI 产品标配。

后续计划：错题本 + 智能组卷推荐、按知识点薄弱做针对性练习。
